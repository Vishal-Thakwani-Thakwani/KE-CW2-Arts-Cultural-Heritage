# LLM Prompts — Competency Question Generation

**Module:** 5CCSAKNE Knowledge Engineering — Coursework 2  
**KE Task:** Competency Question Augmentation (CQs 11–20)  
**Author:** Vishal Thakwani (Domain Expert)

## Overview

The coursework spec requires:
> "The knowledge engineers will augment this list of questions with 10 additional questions generated by LLMs, justifying how the prompts were designed to produce competency questions that are relevant to the domain and complement the first 10 ones you created manually."

This document records all prompts used, the design rationale behind each, and the iterative refinement process.

## Prompt Design Strategy

A **three-stage iterative approach** was used:

1. **Prompt A (Domain Scoping)** — Anchors the LLM in the specific domain, names the ontology classes/properties and the two existing ontologies being extended, and constrains output toward complex SPARQL-answerable query patterns.
2. **Prompt B (Complementarity)** — Feeds the LLM a summary of themes already covered by the 10 manual CQs and explicitly instructs it to avoid overlap while filling specific gaps.
3. **Prompt C (Feasibility Refinement)** — Instructs the LLM to verify each question is answerable by SPARQL over an RDF graph and to remove any subjective/qualitative judgement.

This strategy ensures the generated CQs are (a) grounded in the actual ontology design, (b) genuinely complementary to the manual set, and (c) operationally answerable.

## Prompt A — Domain Scoping and Ontology Alignment

### Prompt Text

> "Act as a knowledge engineer specialising in cultural heritage ontologies. Our group is building a knowledge graph for the Arts and Cultural Heritage domain, covering artworks (paintings, sculptures, prints, drawings, photographs, installations), artists, museums, galleries, exhibitions, conservation events, acquisition events, art periods and movements, mediums, techniques, genres, and awards. We are extending two existing ontologies: CIDOC-CRM (ISO 21127, the international standard for cultural heritage documentation) and the Europeana Data Model (EDM). The core classes include: Artwork (with subtypes Painting, Sculpture, Photograph, Print, Drawing, Installation), Person (Artist, Curator, Conservator, Collector), CulturalInstitution (Museum, Gallery), Place (City, Country, Continent), Event (Exhibition, ConservationEvent, AcquisitionEvent), ArtPeriod, ArtMovement, Medium, Technique, Genre, Award. Key relationships include: createdBy, heldBy, createdDuring, usedMedium, hasGenre, exhibitedIn, locatedIn (transitive), bornIn, activeIn, influencedBy, memberOf, wonAward, restoredArtwork, acquisitionOf. Propose 10 competency questions that this knowledge graph should answer. The questions must be complex enough to require multi-hop SPARQL queries (joining across at least 2–3 entities), and should include at least two questions using aggregation (COUNT, MAX, MIN), at least two involving temporal reasoning (date ranges, ordering), and at least one using negation or absence patterns. The domain is strictly visual arts — exclude music, theatre, dance, and performing arts."

### Design Rationale

This prompt was designed to:
- **Name every class and relationship** from the domain concept sketch so the LLM generates questions grounded in the actual ontology structure, not generic cultural heritage questions
- **Specify CIDOC-CRM and EDM** by name so questions align with the real extensions being implemented
- **Require multi-hop complexity** to ensure CQs go beyond simple lookups (which the manual set already covers)
- **Mandate specific patterns** (aggregation, temporal, negation) to guarantee coverage of SPARQL features not tested by the manual CQs
- **Explicitly exclude performing arts** to respect the domain scope decision documented in domain-expert-report.md

### Result

The LLM generated 10 candidate questions. Several were strong (provenance chains, cross-institution exhibition overlap, negation patterns). Two were too similar to existing manual CQs (one about medium distribution duplicated CQ5's territory, one about awards overlapped CQ4). These were flagged for Prompt B.

## Prompt B — Complementarity and Gap-Filling

### Prompt Text

> "The following themes are already covered by our 10 manually crafted competency questions: (1) artist nationality + creation period filtering, (2) art movement membership + institution holding, (3) conservation event counting, (4) award winners, (5) most common medium before a date, (6) living artist reasoning + geographic containment, (7) transitive geographic location lookup, (8) temporal artwork filtering + institution, (9) multi-genre artists, (10) specific artist + artwork subtype + institution. Generate 10 REPLACEMENT competency questions that DO NOT overlap with any of these 10 themes. Instead, focus on these gaps: cross-institution exhibition comparisons, provenance/ownership chains across countries, art movement temporal span analysis, curatorial decisions across periods, century-level medium distribution changes, negation patterns for unexhibited works, multi-artist collaborative creations, patron/donor networks across museums, and artist influence chains across movements. Each question should be phrased as a natural language question a museum researcher or cultural heritage professional would realistically ask."

### Design Rationale

This prompt was designed to:
- **Summarise each manual CQ's theme** in one phrase so the LLM has a clear exclusion list
- **Name specific gap areas** identified by comparing the manual CQ coverage against the full ontology concept sketch — provenance, curation, patronage, influence, collaboration, and negation are all modelled in the ontology but untested by manual CQs
- **Anchor in real-world use cases** ("museum researcher") to keep questions practical rather than artificial

### Result

All 10 replacement questions avoided manual CQ themes. The provenance question (CQ13) and influence chain question (CQ20) were particularly strong. One question about "most restored artwork" was rephrased in Prompt C because it required subjective ranking.

## Prompt C — SPARQL Feasibility Refinement

### Prompt Text

> "For each of the 10 questions you generated, verify that it can be answered with a SPARQL query over an RDF knowledge graph using only the classes and properties defined above. If a question requires free-text reasoning, subjective judgement, or data not representable in the ontology (e.g., 'most influential', 'most important'), rephrase it to use only structured relationships and quantifiable measures. For example, replace 'most influential artist' with 'artist with the most influencedBy relationships pointing to them'. Also ensure each question involves at least two different classes from the ontology."

### Design Rationale

LLMs tend to generate questions requiring qualitative judgement ("Who is the most significant...", "What is the greatest...") which cannot be resolved by SPARQL alone. This refinement prompt:
- **Forces operationalisation** — every question must be answerable via triple patterns, FILTER, GROUP BY, HAVING, NOT EXISTS
- **Catches class coverage gaps** — requiring 2+ classes per question ensures the CQs actually test the KG's interconnectedness
- **Prevents impossible queries** — any question requiring NLP over free text is excluded since CQs must map to SPARQL

### Result

Two questions were rephrased: "most restored artwork" became the temporal-span restoration query (CQ12), and "most important movement" became the movement temporal span + artist count query (CQ14). Final set of 10 CQs was confirmed.
